# Patterns

`patterns` 记录的是 section 级关系，这是避免 AI 做出“组件堆砌页”的关键层。

## 重点记录的模式

- Hero
- Feature Grid
- Social Proof
- Case Study
- Pricing
- FAQ
- Dashboard Overview
- List + Detail

## 每个 pattern 至少要写

- 适用场景
- 信息优先级
- 推荐布局
- 推荐留白
- 允许的组件组合
- 禁止的构图习惯

## 推荐描述模板

### Hero / Editorial Split

- 适用场景：品牌首页、产品总览、方案介绍
- 信息优先级：标题 > 说明 > 主 CTA > 次 CTA > 信任背书
- 推荐布局：左文本右视觉，或上文本下视觉，不允许完全复刻参考站排布
- 推荐留白：首屏上下间距大于常规 section
- 允许组件：eyebrow、headline、supporting copy、button group、proof strip
- 禁止：
  - 不允许标题、按钮、标签全部居中堆叠
  - 不允许用一整块插画替代信息层级
  - 不允许把参考站 section 顺序原样照搬

## 自检问题

- 页面是否只是堆按钮、卡片、标签，而没有 section 级设计
- 不看颜色和图片，版式关系是否仍有识别度
- 新页面是否已经和参考站拉开了构图差异
