# UI Design Guardrails

本文件用于约束 Claude 在设计或实现 UI 时的行为。

## Brand Mood

- 关键词：[例如：克制、编辑感、可信、轻科技、非销售话术感]
- 目标感受：[例如：像高质量 SaaS 品牌，而不是模板站]
- 避免气质：[例如：廉价营销页、炫技式渐变、泛滥玻璃拟态]

## Non-Negotiables

- 保留整体气质，不保留参考网页构图
- 不允许 1:1 section 顺序复刻
- 不允许复用原站独特插画、icon 语言、品牌措辞
- 必须改变网格、比例、留白节奏、组件边角、描边、阴影策略中的至少 3 项
- 优先使用设计系统中的 semantic tokens 和组件，不允许页面级临时发挥破坏系统

## Token Usage

- 所有颜色、文本、背景、边框优先引用 semantic tokens
- 不允许在实现阶段大面积直接写十六进制颜色
- 同一页面中主强调色只允许有一个主角

## Component Rules

- 先复用现有组件，再新增组件
- 新增组件前，先说明它为何不能由现有组件组合得到
- 同一家族组件必须共享圆角、描边、阴影、状态逻辑

## Layout Rules

- 页面应先确定 section 节奏，再填充组件
- 页面至少使用 2 种层级化留白，不允许全页平均留白
- 不要生成“所有 section 都长一样”的重复页面

## Copy Rules

- 文案克制、清晰、少口号
- 标题强调价值，不堆夸张形容词
- 不使用原参考站品牌表达

## Imagery Rules

- 图片或插画只做辅助，不替代信息结构
- 若没有明确素材，优先用抽象图形、数据图形、材质背景替代
- 不复制参考站插画构图或主题物件

## Delivery Rules

在输出新页面方案时，必须同时给出：

- 使用了哪些 tokens
- 复用了哪些组件
- 套用了哪些 patterns
- 与参考页刻意拉开的 3 个差异点
