# Demo Board Checklist

在进入真实业务页面前，先用这份清单确认演示板是否达标。

## 演示板最低内容

- 色板与 semantic token
- 字阶与字重组合
- 6 到 10 个关键组件
- 2 到 3 套 section pattern
- 1 个完整示例页面

## 系统一致性

- 组件是否都来自同一套圆角、描边、阴影策略
- 不同 section 是否共享同一套排版与留白逻辑
- token 是否能跨组件复用

## 原创性

- 是否已经摆脱参考网页的 section 顺序
- 是否已经摆脱参考网页的构图比例
- 是否已经摆脱参考网页的图形和文案气质

## 可扩展性

- 是否能继续做出至少 3 个页面而不重造系统
- 新页面是否还能保持同一气质
- 另一个设计师或模型接手后，是否能看懂这套规则
