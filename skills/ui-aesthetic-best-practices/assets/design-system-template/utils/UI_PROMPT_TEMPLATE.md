# UI Prompt Template

把下面模板中的方括号内容替换为项目内容，再交给 AI。

```md
你现在基于一个已经整理好的设计系统工作，不是基于原始参考网页直接临摹。

## 项目背景

- 项目名称：[项目名]
- 页面类型：[首页 / 功能页 / 仪表盘 / 活动页 / 登录后页面]
- 目标用户：[谁会看]
- 页面目标：[希望用户完成什么]

## 设计系统输入

- Brand mood: [例如：克制、编辑感、轻科技、可信]
- Foundations: [颜色、排版、间距、圆角、阴影、动效的摘要]
- Semantic tokens: [列出关键 token]
- Components: [列出可复用组件]
- Patterns: [列出可复用 section pattern]

## 参考边界

- 保留气质，不保留构图
- 不允许 1:1 section 顺序复刻
- 不允许复用原站独特插画、icon 语言、品牌措辞
- 必须改变网格、比例、留白节奏、组件边角、描边、阴影策略中的至少 3 项

## 输出要求

请先输出 3 个不同方向的页面方案，每个方案都包含：

1. 页面结构
2. 关键 section 说明
3. 使用到的 components
4. 使用到的 semantic tokens
5. 和参考页拉开的差异点

然后选择你认为最合适的 1 个方向，继续细化为高保真页面描述。

页面必须满足：

- 不像模板站
- 不像参考网页换皮
- 风格统一
- 组件可复用
- 能继续扩展到至少 3 个页面
```

## 追加约束模板

如果你觉得 AI 还在往“熟悉套路”坍缩，可以继续追加：

```md
补充约束：

- 不要默认把 hero 做成居中文案加两颗按钮
- 不要让每个 section 都是卡片宫格
- 不要滥用渐变、玻璃拟态、发光描边
- 优先通过排版、留白、比例建立高级感
- 页面首屏必须具备明确的视觉张力
```
